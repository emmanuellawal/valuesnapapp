# Story 4.5-2: Centralise Breakpoint Constants

**Status:** done  
**Epic:** 4.5 — Retrospective Debt Resolution  
**Origin:** Epic 3 retrospective → re-raised Epic 4 retrospective

---

## Story

**As a** developer working on responsive layout,  
**I want** a single `BREAKPOINTS` constant that all layout components import,  
**So that** changing the desktop breakpoint from 1024 requires modifying one file instead of hunting for literal numbers across the codebase.

---

### What This Story Delivers

**One new file:**
- `apps/mobile/constants/breakpoints.ts` — single source of truth for all UI breakpoints

**Three file modifications:**
- `apps/mobile/app/(tabs)/_layout.tsx` — replace literal `1024` with `BREAKPOINTS.desktop`
- `apps/mobile/app/(tabs)/history.tsx` — replace literals `600`, `1024`, `1440` with `BREAKPOINTS.*`
- `apps/mobile/components/organisms/swiss-sidebar.tsx` — update JSDoc comment to reference the constant

**No changes to logic, no changes to breakpoint values, no new npm packages.**

## Dev Notes

### `constants/breakpoints.ts` — Exact Content to Create

```typescript
/**
 * Responsive breakpoints for layout decisions.
 * Import BREAKPOINTS from this file — do not use literal pixel values in components.
 */
export const BREAKPOINTS = {
  /** Tablet breakpoint — 2-column grid (history) */
  tablet: 600,
  /** Desktop breakpoint — sidebar navigation + 3-column grid */
  desktop: 1024,
  /** Large desktop breakpoint — 4-column grid */
  largeDesktop: 1440,
} as const;
```

### `_layout.tsx` Change (line 22)

**Before:**
```typescript
const isDesktop = width >= 1024;
```

**After:**
```typescript
import { BREAKPOINTS } from '@/constants/breakpoints';
// ...
const isDesktop = width >= BREAKPOINTS.desktop;
```

The import belongs at the top of the file with other imports.

### `history.tsx` Change (lines 55–58)

**Before:**
```typescript
const numColumns =
  width < 600 ? 1 :
  width < 1024 ? 2 :
  width < 1440 ? 3 :
  4;
```

**After:**
```typescript
import { BREAKPOINTS } from '@/constants/breakpoints';
// ...
const numColumns =
  width < BREAKPOINTS.tablet ? 1 :
  width < BREAKPOINTS.desktop ? 2 :
  width < BREAKPOINTS.largeDesktop ? 3 :
  4;
```

### `swiss-sidebar.tsx` Change (JSDoc comment)

Update the JSDoc comment that references `1024px` to reference the constant name instead. The exact line is around line 9 — it currently reads `Renders at width >= 1024px in place of SwissTabBar.` Change it to `Renders at width >= BREAKPOINTS.desktop (1024px) in place of SwissTabBar.`

### Important: Do NOT Import `BREAKPOINTS` Into `swiss-sidebar.tsx` Logic

The `swiss-sidebar.tsx` file does not contain any breakpoint *logic* — the 1024 reference in it is only in a JSDoc comment. Do NOT add an import or any breakpoint logic to that file. Only update the comment text.

### Module Alias

The project uses `@/` as a path alias for `apps/mobile/`. The import `@/constants/breakpoints` will resolve correctly without any tsconfig or metro changes.

### Verification Command

```bash
cd apps/mobile && grep -n "1024\|<600\|< 600\|<1440\|< 1440" app/\(tabs\)/_layout.tsx app/\(tabs\)/history.tsx components/organisms/swiss-sidebar.tsx
```

Should return zero matches after the change.
