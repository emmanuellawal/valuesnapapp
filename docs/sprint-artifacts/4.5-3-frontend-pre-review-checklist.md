## Dev Notes

### Checklist File: Recommended Content

Create `docs/frontend-review-checklist.md` with this content (adjust wording as needed, but keep ≤15 items):

```markdown
# Frontend Pre-Review Checklist

Run through these before marking any frontend story done.
Each item traces to a past finding that reached code review.

## Hooks
- [ ] No `useEffect` + `useFocusEffect` coexisting for the same data fetch — Epic 3.3
- [ ] `useCallback` dependency array is complete (no stale closure risk) — Epic 3.3
- [ ] No conditional hook calls — React Rules of Hooks

## Accessibility
- [ ] All interactive elements (`Pressable`, `TouchableOpacity`) have `accessibilityLabel` — Epic 3.3
- [ ] Non-interactive display elements do NOT use `Pressable`/`TouchableOpacity` — Epic 4.8

## API Boundary
- [ ] If this story touches a transformer, a contract test exercises the real backend response shape — Epics 2–3
- [ ] Mock mode path (`env.useMock`) tested separately from real API path

## Tests
- [ ] Error paths (network failure, cancel, timeout, auth failure) have ≥ 1 test each — Epic 4.4-2
- [ ] New test files run clean in isolation (`jest --testPathPattern=<file>`)

## Constants & Code Hygiene
- [ ] No new literal breakpoint values (600/1024/1440) — use `BREAKPOINTS.*` — Epics 3–4
- [ ] No new magic numbers for timeouts, limits, or animation durations — use named constants
- [ ] No `EXPO_PUBLIC_*` variable contains backend-only credentials — Epic 4.1
```

Add a one-line reference inside the existing `## Key Conventions` section (or create it if absent) of `.github/copilot-instructions.md`:
