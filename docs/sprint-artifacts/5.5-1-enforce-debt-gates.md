# Story 5.5-1: Enforce Debt Gates

Status: done

## Story

As a developer on the ValuaSnap team,
I want a mobile lint script, an extended pre-review checklist, and a mandatory code-review gate,
so that the class of preventable failures that recurred across Epic 5 cannot silently accumulate into Epic 6.

## Background

Three process failures were identified in the Epic 5 retrospective as recurring across multiple stories:

1. **No lint script** — `apps/mobile/package.json` has no `"lint"` script, so `npm run lint` fails in the `mobile` workspace. Lint has never been runnable as a single command from CI or the dev workflow.
2. **Checklist not updated** — The frontend pre-review checklist (`docs/frontend-review-checklist.md`) was created in Story 4.5-3 but was not extended with Epic 5 failure patterns (whitespace guards, assertion depth, helper deduplication).
3. **Code review gaps** — Stories 5-6 and 5-10 both closed without a `## Code Review` section, violating the intent of the review mandate established in Epic 4.

All three gaps are mandatory gates per the Epic 5 retrospective team decision. This story closes them.

**Downstream dependency:** Stories 5.5-3 and 5.5-4 cannot start until this story is done — they run under the lint and checklist gates installed here.

## Dev Notes

### Task A — Lint Script

**ESLint is not installed** in `apps/mobile/` — no `eslint` in `package.json` devDependencies, no config file present. Use Expo CLI to bootstrap everything in one step.

**Step 1 — Bootstrap via Expo CLI** (from `apps/mobile/`, only if `eslint.config.js` does not already exist):
```bash
npx expo lint
```
This command:
- Installs `eslint` and `eslint-config-expo` as devDependencies
- Scaffolds `eslint.config.js` (ESLint v9 flat config)
- Runs lint immediately so you see the first baseline output
- Handles `node_modules`, `.expo`, and build artifact ignores automatically

**If `eslint.config.js` already exists** (e.g. partial bootstrap from a previous attempt): skip this step and run `npm run lint` directly.

**After bootstrap — commit `eslint.config.js`:**
`eslint.config.js` is a source file, not a generated artifact. Verify it is not in `.gitignore` and commit it alongside `package.json`. A missing config in CI will cause lint to fail on the build server even if it passes locally.

**Step 2 — Add script** to `apps/mobile/package.json`, appending to the existing `"scripts"` block:
```json
"lint": "expo lint --max-warnings 0"
```
The `--max-warnings 0` flag is mandatory — without it, `expo lint` exits `0` on warnings and the debt gate silently lets warning drift accumulate. `expo lint --help` confirms the flag passes through to ESLint natively.

Full scripts block for reference (append, do not replace):

**If warnings appear on first run:**
- Fix the code (preferred)
- Or add `// eslint-disable-next-line <rule> -- <reason>` on the offending line
- Never use `/* eslint-disable */` at file level unless the file is auto-generated

Document the final warning count in Completion Notes.

### Task B — Checklist Extension

**File:** `docs/frontend-review-checklist.md`

**Current checklist** has these sections: Hooks, Accessibility, API Boundary, Tests, Constants And Hygiene.

**Add a new section** after `## Tests`:

```markdown
## String And Test Hygiene (Epic 5 patterns)
- [ ] **Whitespace guards:** Every translatable string with dynamic content is trimmed or guarded against accidental leading/trailing whitespace — Epic 5 retro
- [ ] **Assertion depth:** Test assertions are specific enough to fail if the target value changes — no bare `.toBeTruthy()` where `.toBe('exact value')` is possible — Epic 5 retro
- [ ] **Helper deduplication:** `getTextContent`, `findByText`, or similar test helpers are defined once and imported, not re-implemented inline — Epic 5 retro
- [ ] **Code review section present:** The story file includes a `## Code Review` section before being moved to `done` — Epic 5 retro (mandatory gate)
```

The last bullet is the mandatory code-review gate. It applies to every story from 5.5-1 onwards.

**Important:** All four items in this section are **manual pre-review checks**, not enforced by the lint script. `expo lint` catches static analysis issues (types, imports, rules). Whitespace guards, assertion depth, helper deduplication, and code review presence are human-verified at story close-out, not at CI time.
